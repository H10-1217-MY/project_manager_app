# 📂 Project Manager App

Project Manager App は、サーバやDBを使わずに、共有フォルダ上でプロジェクト情報と関連ファイルを管理する Windows 向けデスクトップアプリです。

各PCではローカルアプリを起動し、共通の共有フォルダに保存された JSON とプロジェクトファイルを参照します。  
APIサーバやDBを用意できない環境でも、プロジェクト単位でファイル・説明・ステータス・更新履歴を管理できることを目的としています。

---

## 🧭 概要

ローカルまたは共有フォルダ上のプロジェクト関連ファイルを、プロジェクト単位で登録・閲覧・管理するためのアプリケーションです。

プロジェクトごとに専用フォルダを作成し、関連ファイルと `metadata.json` を保存します。  
プロジェクト一覧は `index.json` で管理し、アプリ上から検索・閲覧・編集できます。

DBやAPIサーバを使わず、ファイルシステムとJSONだけで管理できるため、サーバを立てられない環境でも利用できます。

- 🕘 更新履歴の記録・表示

  - 説明文検索
- 📄 関連ファイル一覧の表示
- 📥 ファイル取得
- 📦 プロジェクト一式のZIP出力

## 🗂️ 共有フォルダ側の構成

アプリで設定した共有フォルダには、以下のような構成でデータが保存されます。

```text
C:/server/
├── index.json
├── index.json.bak
├── projects/
│   ├── PJT_YYYYMMDD_HHMMSS_xxxx/
│   │   ├── metadata.json
│   │   ├── metadata.json.bak
│   │   └── files/
│   │       └── 登録されたファイル・フォルダ
│   └── ...
└── trash/
    └── 削除されたプロジェクト
```

## ⚙️ 設定項目

アプリ内の設定画面から、以下を指定します。

| 項目             | 内容                                         |
| ---------------- | -------------------------------------------- |
| 共有フォルダパス | プロジェクト情報や関連ファイルを保存する場所 |
| 既定の保存先     | ファイル取得やZIP出力時の初期保存先          |

例:

```json

  "default_download_path": "C:/Users/user/Downloads/ProjectHub"
}
```

## 🚀 起動方法

Python環境のある方は、プロジェクトルートで以下を実行します。

```bash
python main.py
```

Python環境のない方は、`dist` フォルダの中にある `ProjectManager.exe` を使用してください。

---

## 🔄 基本的な使い方

### 1. 共有フォルダを設定する

初回起動後、設定画面から共有フォルダを指定します。

### 2. プロジェクトを登録する

登録画面から以下を入力します。

- プロジェクト名
- 説明文
- 関連ファイル
- 関連フォルダ

登録すると、共有フォルダ内にプロジェクトID付きのフォルダが作成され、`metadata.json` と関連ファイルが保存されます。

### 3. プロジェクトを閲覧する

閲覧画面では、以下を確認できます。

- プロジェクト名
- 説明文
- ステータス
- 作成日
- 更新日
- 関連ファイル一覧
- 更新履歴

### 4. ファイルを取得する

閲覧画面でファイルを選択し、保存先を指定してコピーできます。

### 5. ZIP出力する

選択したプロジェクト一式をZIPファイルとして出力できます。

### 6. プロジェクトを削除する

削除したプロジェクトは完全削除されず、共有フォルダ内の `trash` フォルダへ移動されます。

## 🧭 運用ルール

このアプリは、Gitのような差分管理ツールではなく、プロジェクトの情報と関連ファイルを共有・整理するための軽量な管理ツールです。

そのため、登録済みプロジェクトのファイル構成などの中身が大きく変わった場合は、既存プロジェクトを直接差し替えるのではなく、新しいプロジェクトとして登録する運用を推奨します。

例:

```text
イベント収集ツール_v1
イベント収集ツール_v2
イベント収集ツール_提出版
イベント収集ツール_20260430
```

ファイルの詳細な差分確認やバージョン管理は、Gitなど専用ツールの利用を想定しています。

## 🛡️ 登録時の除外対象

フォルダ登録時、不要になりやすい以下のフォルダやファイルはコピー対象から除外します。

### 除外フォルダ

```text
.git
.venv
venv
__pycache__
.pytest_cache
.mypy_cache
node_modules
build
dist
```

### 除外ファイル

```text
*.pyc
*.pyo
Thumbs.db
.DS_Store
```

これにより、Git管理フォルダや仮想環境、キャッシュファイルを登録対象から外し、登録処理を軽量化します。

## 🧠 開発意図

このアプリは、サーバやDBを用意できない環境でも、プロジェクト情報と関連ファイルを整理・共有できるようにすることを目的として作成しました。

単なるフォルダ管理ではなく、プロジェクト単位で以下をまとめて扱えるようにしています。

- 何のプロジェクトか
- どのような状態か
- どのファイルが関係しているか
- いつ更新されたか
- どのような履歴があるか

これにより、個人開発・小規模チーム・共有フォルダ運用の現場で、プロジェクト情報を見失いにくくすることを目指しています。
